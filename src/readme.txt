Стартиране на проект:

При стартиране на проекта, ще се изведе текст, обясняващ на потребителя в какъв режим иска да пусне програмата:
1. Compress from a file - ако изберете този режим, то тогава програмата ще очаква име на файл, който ще компресира.
2. Decompress from a file - ако изберете този режим, то тогава програмата ще очаква име на файл, който ще декомпресира. Важното тук е че има конвенция за имената на тези файлове. Те трябва да звършват с _compressed.txt задължително!!!

Когато съответната компресия/декомпресия завърши ще се изведе процента на компресия, тоест съотношението на размера на компресирания файл върху размера на входящия файл.

Изходни документи:

{file name}_additional.txt - в този файл се съхраняват сериализираното дърво на Хъфман на съответния низ от {file name}.txt и едно число, отговарящо на добавените 0-ли в бинарния код на компресирания низ.
{file name}_decimal.txt - в този файл се съдържа числовата последователност на компресирания низ (на всеки байт отговаря едно число в интервала 0 до 255)
{file name}_binary.txt - в този файл се съдържа бинарната последователност на компресирания низ
{file name}_compressed.txt - този файл съдържа истинският компресиран низ
{file name}_decompressed.txt - този файл съдържа декомпресирания низ в резултат от декомпресирането
